Задача: создать систему процедурной генерации карт, аналогичную Brawl Stars, с соблюдением строгих правил играбельности, баланса и корректного размещения объектов.

Карта должна генерироваться случайно, но при этом всегда оставаться логичной, проходимой и честной для игроков.

-----------------------------------
0. ИСПОЛЬЗОВАНИЕ АССЕТОВ
-----------------------------------
- Использовать только загруженные текстуры и 3D модели
- Не создавать новые ассеты
- Не изменять стиль
- Все объекты должны соответствовать grid

-----------------------------------
1. ОСНОВА: GRID-СИСТЕМА
-----------------------------------
- Карта = 2D массив клеток (TileMap)
- Каждая клетка имеет фиксированный размер
- Все объекты размещаются строго по grid

-----------------------------------
2. ТИПЫ КЛЕТОК
-----------------------------------
0 = пусто (проходимо)
1 = стена (непроходимо)
2 = куст (скрытие)
3 = вода (непроходимо)
4 = объект (ящик и т.д.)
5 = зона спавна
6 = цель (кристаллы / сейф / база)

-----------------------------------
3. ГАРАНТИЯ ПРОХОДИМОСТИ (КРИТИЧНО)
-----------------------------------
- На карте НЕ должно быть тупиковых зон, из которых нельзя выйти
- Все основные зоны должны быть связаны

Обязательно:
- Проверка пути (pathfinding)
- Из любой точки спавна должен быть путь:
  → к центру карты
  → к целям (кристаллы, сейф и т.д.)

Если путь отсутствует:
→ карта пересоздается

-----------------------------------
4. ЗАПРЕТ ЗАКРЫТЫХ ЗОН
-----------------------------------
НЕЛЬЗЯ:
- создавать полностью замкнутые области
- окружать игрока стенами/водой без выхода
- создавать "карманы", где игрок застрянет

-----------------------------------
5. ПРАВИЛА СПАВНА
-----------------------------------
- Спавн должен находиться в проходимой зоне
- Вокруг спавна должно быть свободное пространство (минимум 2-3 клетки)
- Нельзя:
  - ставить стены вплотную к спавну
  - блокировать выход из спавна
  - перекрывать кустами или водой

-----------------------------------
6. ПРАВИЛА ОБЪЕКТОВ
-----------------------------------
- Текстуры и модели НЕ могут:
  - пересекаться друг с другом
  - накладываться
  - занимать одну клетку одновременно

- Каждый объект = отдельная клетка или группа клеток

-----------------------------------
7. ЗАПРЕТ ПЕРЕКРЫТИЯ ВАЖНЫХ ЗОН
-----------------------------------
НЕЛЬЗЯ размещать:
- стены
- воду
- кусты

на:
- спавн
- центральную точку (кристаллы)
- сейф
- базу

Эти зоны должны быть:
- чистыми
- доступными
- визуально читаемыми

-----------------------------------
8. БАЛАНС КАРТЫ (КРИТИЧНО)
-----------------------------------
- Карта должна быть симметричной или сбалансированной
- Ни одна сторона не должна иметь преимущество

Примеры:
- одинаковые расстояния до центра
- равные укрытия
- равный доступ к объектам

-----------------------------------
9. РЕЖИМЫ И ПРАВИЛА
-----------------------------------

A) СТОЛКНОВЕНИЕ (Showdown):
- большая карта
- разбросанные укрытия
- нет замкнутых зон
- свободное перемещение по всей карте

B) ЗАХВАТ КРИСТАЛЛОВ:
- центр карты открыт
- точка кристаллов в центре
- симметричный доступ с обеих сторон

C) ВЫНОС КРИСТАЛЛОВ:
- центр → сбор
- база → доставка
- должны быть альтернативные пути (не один коридор)

D) ОГРАБЛЕНИЕ:
- у каждой команды есть сейф
- сейф не перекрыт объектами
- есть несколько путей к сейфу

E) ОСАДА:
- база и зона робота
- открытые пути атаки
- нет узких "бутылочных горлышек"

-----------------------------------
10. СТРУКТУРА КАРТЫ
-----------------------------------
Карта должна содержать:
- открытые зоны (для боя)
- укрытия (стены)
- зоны скрытия (кусты)
- проходы (не узкие)

Баланс:
- не слишком открыто
- не слишком закрыто

-----------------------------------
11. ИЗБЕГАНИЕ ХАОСА
-----------------------------------
Запрещено:
- случайное нагромождение объектов
- хаотичное размещение
- отсутствие логики структуры

Карта должна выглядеть:
→ как спроектированная, а не случайная

-----------------------------------
12. КОНТРОЛЬ КАЧЕСТВА (ОБЯЗАТЕЛЬНО)
-----------------------------------
После генерации выполнить проверки:

1. Есть ли путь между всеми ключевыми зонами?
2. Есть ли заблокированные области?
3. Есть ли перекрытия объектов?
4. Есть ли зазоры или ошибки grid?
5. Сбалансирована ли карта?

Если хоть одно условие не выполнено:
→ пересоздать карту

-----------------------------------
13. ЦЕЛЬ
-----------------------------------
Генерировать карты, которые:
- полностью проходимы
- сбалансированы
- визуально чистые
- без зазоров и пересечений
- соответствуют игровым режимам

Карта должна выглядеть как сделанная дизайнером,
а не как случайный набор объектов.
